# July 2026

## Security hardening

- Removed command-line AWS credential examples and required SDK/OIDC credential-provider usage.
- Made deploy authorization fail closed through a private S3 object with exact IAM role ARNs and explicit scopes.
- Rejected publish-on-create and stopped draft upsert from mutating public alias metadata or lookup records.
- Added strict canonical validation for domains, version IDs, and domain-rooted JSON paths before storage.
- Restricted S3 writes to sites/* and kept the authorization object read-only.
- Routed pull-request branch names through workflow environment variables before Bash evaluation to prevent expression injection.
- Added regression coverage for account boundaries, malformed authorization rules, traversal and platform path hazards, repository hygiene, and environment-specific publication.
- Replaced repository-root Lambda packaging with a deterministic two-file `CodeUri` allowlist and made CI/deploy workflows verify the exact SAM artifact before configuring AWS credentials.
- Made the deploy jobs provide their environment region to credential-free SAM validation and build steps, while keeping AWS credential configuration after those gates.

## Server-only integration infrastructure — Phase 1

- Added four generic, closed server-policy contracts for data spaces, commerce, integration bindings, and notification policies while preserving the two legacy compatibility descriptors.
- Bound every accepted authorization rule and server descriptor to one exact environment, tenant, draft, and domain scope; ambiguous or rebound scopes fail closed.
- Added strict limits and semantic checks for descriptor size/depth, PII, secrets, provider resource identifiers, capabilities, subscriptions, editable prices, inventory, shipping, fiscal approvals, and notification references.
- Added deterministic notification secret-name derivation and metadata-only `DescribeSecret` preflight with exact lifecycle and ownership tags; no secret values are read or returned.
- Made package versions single-use and immutable with conditional S3 writes, an exact all-object inventory, a sorted SHA-256 manifest, stored-package revalidation, and expected-revision DynamoDB pointer updates.
- Removed AWS `dev` deployment configuration. Only explicit `test` and `production` stacks remain; local development does not provision AWS resources.
- Expanded the reproducible Lambda artifact from two to exactly seven allowlisted files: three Python modules and four schemas.
- Split deployment validation from deployment identity: tests, SAM lint/build, and artifact upload run without OIDC; only the dependent deploy job receives OIDC and re-verifies the downloaded artifact before AWS credentials are configured.
- Added a non-packaged operator bootstrap that derives the canonical per-draft scope registry, verifies GitHub environment variables and exact IAM OIDC trust, generates ARN-only environment authorization, and performs reviewed conditional/versioned S3 writes with exact readback and copy-forward rollback.
- Made onboarding append-safe: apply now binds the reviewed current scope ETag/version/SHA-256, permits only create/idempotent/strict append transitions, writes scope before authorization, and rejects removals or mapping/ID changes.
- Bound test approval evidence to exact workflow identities, endpoint, stack, current scope/authz object versions and hashes, canary timing, unsigned denial, workflow artifact bytes, source commit, and deployed Lambda code hash/revision; production re-collects the evidence and rejects later drift.
- Replaced change-set discovery with one deterministic per-run name. Both deploy workflows package normalized allowlisted bytes under a unique run prefix in the environment's private payload bucket, describe and validate only the returned ARN, reject removals/replacements, execute only that ARN, and retain workflow artifacts for seven days.
- Added the `FunctionArn` stack output and a production parity gate that compares the live test and production Lambda `CodeSha256`; the production deploy role requires only exact read access to the test stack outputs.
- Closed legacy compatibility to three exact domain/file/canonical-hash triples verified against canonical and currently referenced test/production copies; every new or modified legacy shape now fails closed while recursive sensitive-value scanning remains defense in depth.
- Added checksum-valid CLABE and Luhn PAN detection across every server descriptor, including generic field names, without treating digit length alone as sufficient evidence.
- Required promotion merge trees to equal their exact source-tip trees. Production now consumes the immutable exact successful test-run SAM artifact and manifest, proves the current live test Lambda ZIP and the exact run-scoped production package ZIP share its contents and `CodeSha256` before exposure, rechecks the live test hash immediately before change-set execution, and retains post-deploy parity.
- Bound canary endpoint evidence to GitHub's environment-variable `updated_at < run.created_at`; equality fails closed because timestamp resolution cannot prove ordering, while exact run-artifact endpoint capture remains a future cross-repository hardening step.
- Documented the unbounded unique S3 packaging-prefix residual (about 0.116 GB per 1,000 runs at the current raw seven-file size before compression); lifecycle remains a separate measured cost/rollback decision.
- Moved the new ARN-and-scope authorization contract to the parallel `system/deploy-authz-v2.json` key with conditional create/readback, leaving legacy `system/deploy-authz.json` unchanged for the currently deployed role-name runtime and protected rollback.

# July 2026

## Security hardening

- Removed command-line AWS credential examples and required SDK/OIDC credential-provider usage.
- Made deploy authorization fail closed through a private S3 object with exact IAM role ARNs and explicit scopes.
- Rejected publish-on-create and stopped draft upsert from mutating public alias metadata or lookup records.
- Added strict canonical validation for domains, version IDs, and domain-rooted JSON paths before storage.
- Restricted S3 writes to sites/* and kept the authorization object read-only.
- Added regression coverage for account boundaries, malformed authorization rules, traversal and platform path hazards, repository hygiene, and environment-specific publication.
